# Git Commit

Create a git commit following this project's conventions.

## Commit Message Rules

- **Language**: Always write commit messages in English
- **Format**: Imperative mood subject line + optional detailed body
- **Subject line**: `<Action verb> <what changed>` (max ~72 chars)
- **Action verbs**: Add, Fix, Update, Remove, Rename, Configure, Refactor, Implement
- **Body**: Bullet points with `-` prefix describing specific changes per file/component

## Commit Message Examples

Single-purpose change:
```
Fix e2e test failure: enable webhook and cert-manager in kustomization
```

Multi-component change:
```
Add init container fix, monitoring, multiPodPerHost, and Cilium integration

- internal/podutil/container.go: Init container with volume initialization
- internal/controller/reconciler_monitoring.go: Prometheus sidecar injection
- api/v1alpha1/types_pod.go: MultiPodPerHost field with anti-affinity
- internal/controller/reconciler_networkpolicy.go: CiliumNetworkPolicy support
```

Test addition:
```
Add comprehensive unit tests for all four enhancements

- container_test.go: BuildInitContainer tests
- pod_test.go: Anti-affinity injection and sidecar tests
- webhook_test.go: Monitoring defaults and CE constraint validation
- Coverage: api/v1alpha1 0%->15.6%, configgen 64.3%->71.9%
```

## Workflow

1. Run `git status` and `git diff --staged` to inspect current changes
2. If nothing is staged, stage the relevant files (prefer specific files over `git add .`)
3. Never stage files that contain secrets (`.env`, credentials, tokens)
4. Never stage auto-generated files unless the generator was explicitly run:
   - `zz_generated.deepcopy.go` (generated by `make generate`)
   - `config/crd/bases/*.yaml` (generated by `make manifests`)
   - `go.sum` (generated by `go mod tidy`)
5. Draft the commit message based on the actual changes
6. Create the commit
7. Run `git status` to verify success

## Pre-commit Verification

Before committing, consider if these should be run (ask user if unclear):
- `make fmt` - Format Go code
- `make vet` - Static analysis
- `make lint` - Lint check
